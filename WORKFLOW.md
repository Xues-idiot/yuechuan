# 阅川 (YueChuan) - 自主工作流

> **核心定位**：多平台内容聚合 + AI消化 + 直接消费
> **杀手锏**：视频转文字 + 沉浸式翻译 + 知识图谱

---

## 零、产品定位

### 阅川是什么
**中文互联网多平台内容聚合 + AI智能阅读器** - 不是简单的RSS阅读器，是帮你把内容读完、整理好、精华直接给你看

### 阅川的竞争力
**完整的内容消费闭环**：订阅 → 抓取 → 转录/翻译 → AI整理 → 知识图谱

vs Folo：Folo 是英文的、无视频转录；**阅川是中文优先的完整pipeline**
vs Readwise：Readwise 是封闭的、只做高亮同步；**阅川是开源的、做完整内容处理**

### 阅川的杀手锏
**视频转文字 + 沉浸式翻译 + 知识图谱**：
- 视频内容直接转文字阅读
- 外语内容实时翻译成中文
- 知识点自动关联，形成知识网络

## 核心原则：作品集标准

> 核心理念：像 Sigma 学习，自主循环、自主进化、持续改进
> **但更重要的是：做出能拿出手给面试官看的作品**

**作品标准**：
1. 后端Agent能正常工作（调用API有返回）
2. 前端有完整的用户界面（不是空白页面）
3. 前后端能联调（不是各自独立）
4. 能拿出手给面试官看（不是半成品）

**检查清单 - 每实现一个功能必须同时完成**：
- [ ] 后端：Agent代码 + API接口
- [ ] 前端：操作界面 + 状态展示
- [ ] 联调：前后端能正常通信
- [ ] 体验：加载状态、错误处理
- [ ] 演示：功能可以实际运行展示

**必须思考的问题**（不只是完成功能）：
1. **阅川的核心竞争力是什么？** 比Folo/Readwise 好在哪儿？
2. **数据从哪来？** 各平台内容怎么获取？RSSHub够用吗？
3. **为什么要做这个功能？** 是因为参考项目有这个，还是真的有需求？
4. **竞品会怎么评价？** 面试官问"和 Folo 有什么区别"怎么答？

**常见错误**：
- ❌ 只做后端，不管前端
- ❌ 前端只有一个空白页面
- ❌ 前后端不通信，各玩各的
- ❌ 功能实现了但界面无法演示
- ❌ 觉得"能跑通就行" - **作品级标准是"能拿出手"**
