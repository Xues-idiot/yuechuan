# 阅川 (YueChuan) - 多平台内容聚合阅读器

> **口号**：阅读如江河，内容汇聚
> **核心价值**：订阅万物 → AI消化 → 直接消费

---

## 一、产品定位

### 阅川是什么

**中文互联网多平台内容聚合 + AI智能阅读器**

不是简单的RSS阅读器，也不是视频转文字工具，而是**帮你把各平台内容读完、整理好、精华直接给你看**的工具。

### 核心场景

```
早上打开阅川
    ↓
看到订阅源更新（微信公众号/B站/小红书/抖音/YouTube...）
    ↓
点开任意内容
    ├── 图文 → 直接阅读 + AI辅助摘要
    ├── 视频 → AI转录成文字 + 摘要 + 翻译
    └── 外语 → 沉浸式翻译成中文
    ↓
不用自己去每个平台刷了
直接消费整理好的内容
```

### 解决什么问题

| 痛点 | 阅川解决 |
|------|----------|
| 太多平台要刷 | 统一订阅入口 |
| 视频太长不想看 | AI转文字，直接读 |
| 外语内容看不懂 | 沉浸式翻译 |
| 内容太多看不过来 | AI摘要提炼精华 |
| 看完就忘 | 知识图谱关联 |

## 二、核心功能

### F0 - 杀手锏（必须完美）

1. **多平台订阅**
   - 微信公众号（RSSHub）
   - 小红书（图文+视频）
   - 哔哩哔哩（视频+专栏）
   - 抖音（视频）
   - YouTube（视频+字幕）
   - 微博
   - 知乎
   - 即刻/小宇宙

2. **视频AI转录**
   - Whisper转录（自托管或API）
   - 支持中文/英文/日文等多语言
   - 转录后可编辑

3. **AI摘要整理**
   - 一句话概括内容
   - 关键点提炼
   - 自动生成标签/分类

4. **沉浸式翻译**
   - 调用沉浸式翻译API
   - 支持10+翻译引擎
   - 双语对照阅读

5. **知识图谱**
   - 知识点自动关联
   - 阅读历史串联
   - 发现相关内容的阅读

### F1 - 重要功能

6. **阅读进度同步**
   - 多设备同步
   - 断点续读

7. **收藏与笔记**
   - 高亮标记
   - 笔记导出

8. **定时推送**
   - 每日摘要推送
   - 自定义规则

### F2 - 扩展功能

9. **间隔复习**
   - 类似Readwise的间隔复习
   - 重要内容定期提醒

10. **导出到笔记工具**
    - Obsidian
    - Notion
    - 印象笔记

## 三、竞品对比

| 竞品 | 优点 | 缺点 |
|------|------|------|
| **Folo** | 多平台聚合、AI摘要 | 英文界面、免费次数少、无视频转录 |
| **Readwise** | 高亮同步、间隔复习 | 封闭、不支持视频、订阅贵 |
| **RSSHub** | 可生成任何RSS | 只是RSS、不做内容处理 |
| **沉浸式翻译** | 翻译质量好 | 只是翻译、不做聚合 |
| **Echo** | 视频转录能力强 | 单内容研究、不是聚合消费 |

### 阅川的差异化

1. **中文优先** - 全中文界面，中国平台深度支持
2. **视频转录** - Folo没有的
3. **聚合+AI** - 不是单一功能，是完整pipeline
4. **免费开源** - 可自托管

## 四、技术架构

### 核心模块

```
yuechuan/
├── backend/
│   ├── agents/              # AI Agent
│   │   ├── summarizer.py    # 摘要生成
│   │   ├── transcriber.py   # 视频转录
│   │   └── translator.py    # 翻译
│   ├── services/
│   │   ├── rss_processor.py # RSS处理
│   │   ├── platform_fetcher.py # 各平台内容获取
│   │   └── knowledge_graph.py  # 知识图谱
│   └── api/
│
├── frontend/
│   ├── src/app/            # Next.js页面
│   ├── components/         # 组件
│   └── ...
│
├── shared/                 # 共享AI能力（可与其他项目共用）
│   └── ai-core/
│
└── _reference/            # 参考项目
    ├── FreshRSS/
    ├── RSSHub/
    ├── v2/ (Miniflux)
    └── deer-flow/
```

### 关键技术选型

| 环节 | 技术 |
|------|------|
| 前端 | Next.js + React + shadcn/ui |
| 后端 | Python/FastAPI + LangGraph |
| 视频转录 | Whisper (本地) / 第三方API |
| 翻译 | 沉浸式翻译API / DeepL / GPT |
| 数据库 | PostgreSQL + ChromaDB (知识图谱) |
| 订阅同步 | RSSHub + 官方API |
