# 产品迭代工作流

## 1. 迭代循环

```
功能 → 优化 → Bug修复 → 功能（无限循环）
每50轮进行一次功能审查
```

---

## 2. 10步进化循环

### Step 1-4: 准备阶段

1. **读取 README.md、PROGRESS.md 确认定位**
2. **读取学习资源**
3. **并行学习** (GitHub + 业务知识)
4. **自我检查**

### Step 5-6: 开发阶段

5. **运行 `npm run build` 验证构建**
6. **实现改进** (代码层面 + 业务层面)

### Step 7-10: 收尾阶段

7. **构建验证**
8. **更新 PROGRESS.md 文档**
9. **GitHub提交** (`git add -A && git commit && git push`)
10. **更新记忆系统** (Claude Code Memory)

## 5. 工作流程

### 每次迭代执行

```
1. 读取当前进度 (PROGRESS.md)
2. 确定本轮任务
3. 执行开发
4. npm run build 验证
5. git commit + push
6. 更新 PROGRESS.md
7. 更新记忆系统
8. 报告状态
```

### 提交规范

```
✨ feat: 功能描述
🐛 fix: 修复描述
⚡ optimize: 优化描述
🔧 refactor: 重构描述
📝 docs: 文档更新
```

---

## 6. 循环检查点

每次迭代自问：

1. 本轮完成了什么？
2. 下一个要做什么？
3. 有没有新问题需要记录？
4. 记忆系统是否需要更新？
5. 达到50轮了吗？（需要功能审查）
