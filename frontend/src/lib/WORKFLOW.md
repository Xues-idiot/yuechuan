# 产品迭代工作流

## 1. 迭代循环

```
功能 → 优化 → Bug修复 → 功能（无限循环）
每50轮进行一次功能审查
```

---

## 2. 10步进化循环

### Step 1-4: 准备阶段

1. **读取 README.md、PROGRESS.md 确认定位**
2. **读取学习资源**
3. **并行学习** (GitHub + 业务知识)
4. **自我检查**

### Step 5-6: 开发阶段

5. **运行 `npm run build` 验证构建**
6. **实现改进** (代码层面 + 业务层面)

### Step 7-10: 收尾阶段

7. **构建验证**
8. **更新 PROGRESS.md 文档**
9. **GitHub提交** (`git add -A && git commit && git push`)
10. **更新记忆系统** (Claude Code Memory)

---

## 3. 功能迭代四象限

每次迭代需要平衡以下四类任务：

```
┌─────────────────────────────────────────────────────────┐
│                    功能迭代四象限                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌─────────────────┬─────────────────┐               │
│   │   🔍 探索学习    │   ✨ 添加功能    │               │
│   │   学习新项目     │   新功能实现     │               │
│   │   参考竞品       │                 │               │
│   ├─────────────────┼─────────────────┤               │
│   │   🐛 Bug修复    │   ⚡ 代码优化    │               │
│   │   修复问题      │   重构 & 性能    │               │
│   └─────────────────┴─────────────────┘               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**每轮迭代至少涉及 1-2 个象限：**
- 探索学习：搜索类似项目，学习新功能/技术
- 添加功能：根据产品需求实现新功能
- Bug修复：修复已知问题
- 代码优化：重构、性能、代码质量

---

## 4. 探索学习流程

### 4.1 搜索类似项目
- 搜索 GitHub Trending 相关技术栈
- 搜索 Product Hunt 相似产品
- 搜索 CSDN/掘金 相关实现

### 4.2 学习新功能
- 查阅官方文档
- 分析最佳实践
- 评估适用性

### 4.3 记录学习成果
- 更新到 MEMORY.md
- 建议可应用到当前项目的功能

## 7. 工作流程

### 每次迭代执行

```
1. 确定本轮任务（四象限）
2. 执行开发/学习/优化
3. npm run build 验证
4. git commit + push
5. 更新记忆系统
6. 报告状态
```

### 提交规范

```
✨ feat: 功能描述
🐛 fix: 修复描述
⚡ optimize: 优化描述
🔧 refactor: 重构描述
📝 docs: 文档更新
🔍 explore: 探索学习
```

---

## 8. 循环检查点

每次迭代自问：

1. 本轮完成了什么？（哪个象限）
2. 下一个要做什么？
3. 有没有新问题需要记录？
4. 记忆系统是否需要更新？
5. 达到50轮了吗？（需要功能审查）
