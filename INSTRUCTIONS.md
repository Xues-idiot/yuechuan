# 阅川 - 产品说明

> 本文档供 AI Agent 阅读，理解阅川是什么、要做什么

---

## 一、阅川是谁

**阅川**是一个中文互联网多平台内容聚合 + AI智能阅读器。

核心理念：**帮你把内容读完、整理好、精华直接给你看**

---

## 二、解决什么问题

### 用户的痛苦

1. **太多平台要刷**
   - 微信公众号要看
   - 小红书要看
   - B站要看
   - 抖音要看
   - YouTube要看
   - 微博、知乎、即刻...
   - 根本刷不过来！

2. **视频太长了**
   - 一个视频30分钟
   - 但其实就想知道讲了什么
   - 看完发现没干货，浪费时间

3. **外语看不懂**
   - YouTube有好多好视频
   - 但英文听不懂
   - 生肉啃不动

4. **看完就忘**
   - 当时看的时候觉得有道理
   - 第二天全忘了
   - 没有形成自己的知识体系

## 三、阅川怎么用

### 场景1：早上通勤

```
打开阅川
    ↓
看到订阅源有10篇更新
    ↓
先看 AI 摘要，挑感兴趣的
    ↓
点进去，图文直接读
    ↓
视频 → AI转文字，当文章读
    ↓
英文 → 沉浸式翻译，当中文读
    ↓
30分钟刷完平时2小时的内容
```

### 场景2：发现好内容

```
看到一篇好文章
    ↓
高亮标记精彩段落
    ↓
AI 自动关联相关知识
    ↓
存入知识图谱
    ↓
后续相关阅读自动推荐
```

## 四、核心功能

### F0 - 杀手锏（必须完美）

| 功能 | 说明 |
|------|------|
| **多平台订阅** | 微信/小红书/B站/抖音/YouTube/微博/知乎/即刻 |
| **视频AI转录** | Whisper转文字，支持中文/英文/日文 |
| **AI摘要** | 一句话概括 + 关键点提炼 |
| **沉浸式翻译** | 外语内容实时翻译 |
| **知识图谱** | 知识点自动关联 |

### F1 - 重要功能

| 功能 | 说明 |
|------|------|
| **阅读进度同步** | 多设备同步，断点续读 |
| **收藏与笔记** | 高亮标记，笔记导出 |
| **定时推送** | 每日摘要推送到手机 |

### F2 - 扩展功能

| 功能 | 说明 |
|------|------|
| **间隔复习** | 类似Readwise，定期提醒复习重要内容 |
| **导出到笔记工具** | Obsidian / Notion / 印象笔记 |

## 五、技术限制

### 平台限制

| 平台 | 获取方式 | 限制 |
|------|----------|------|
| 微信公众号 | RSSHub | 只能获取摘要，全文需登录 |
| 小红书 | RSSHub / 第三方 | 部分内容可获取 |
| 哔哩哔哩 | 官方API | 视频可获取，有字幕 |
| 抖音 | 第三方 | 限制较多 |
| YouTube | 官方API | 有字幕可直接用 |

### AI 限制

| 功能 | 限制 |
|------|------|
| 视频转录 | 长视频需要较长处理时间 |
| 翻译 | 依赖第三方API，有配额限制 |
| 摘要 | 生成质量依赖模型能力 |

## 七、回答面试官

**Q: 和 Folo 有什么区别？**
A: Folo 是英文的，我们中文优先；Folo 不能转录视频，我们可以；我们是开源的，Folo 是闭源的。

**Q: 和 Readwise 有什么区别？**
A: Readwise 专注高亮同步，我们是完整的内容消费 pipeline；Readwise 订阅贵，我们是开源免费的。

**Q: 视频转录怎么做的？**
A: 使用 Whisper 模型，可以本地部署也可以用云端 API。

**Q: 知识图谱怎么实现的？**
A: 使用 ChromaDB 向量数据库，内容embedding后自动关联相似内容。
